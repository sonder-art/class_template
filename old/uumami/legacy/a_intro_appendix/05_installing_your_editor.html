What is a Code Editor?

A code editor is your digital workshop, your primary tool for creating. It‚Äôs like a word processor (like Microsoft Word or Google Docs) but supercharged for writing code. It understands programming languages, helping you with color-coding, auto-completion, and finding errors.

Step 1: Install Cursor

We recommend using your system‚Äôs package manager from the terminal, as it‚Äôs the fastest and most reliable way to install and update software. Choose the tab for your operating system.

  • macOS (Homebrew)
  • Windows (Winget)
  • Linux (AppImage)

  1. Requirement: You need Homebrew, the package manager for macOS. If you didn‚Äôt install it in the Python step, open your Terminal and follow the instructions on their website.

  2. Install Cursor: Run this command in your terminal:

    brew install --cask cursor

  1. Requirement: This uses the built-in winget package manager, which is standard on modern Windows.

  2. Install Cursor: Open PowerShell as an administrator and run:

    winget install Anysphere.Cursor

The most universal method for Linux is using the AppImage.

  1. Go to the official Cursor website and download the .AppImage file.
  2. Open your terminal and navigate to your Downloads folder: cd ~/Downloads
  3. Make the file executable: chmod +x Cursor-*.AppImage
  4. Run it: ./Cursor-*.AppImage --no-sandbox

Step 2: A Quick Tour of the Interface

In software development, you don‚Äôt open individual files; you open the entire project folder. After you clone the class project in a later step, you will open that folder in Cursor.

When you do, you‚Äôll see three main areas:

  1. File Explorer (Left): A tree view of all the files and folders in your project. Click any file here to open it.
  2. Editor Pane (Center): This is where your files open in tabs. You‚Äôll do all your writing and editing here.
  3. Terminal Panel (Bottom): A command line built directly into Cursor. You can open it by pressing Ctrl+` (the backtick key, next to 1) or from the ‚ÄúTerminal‚Äù > ‚ÄúNew Terminal‚Äù menu. We will use this constantly.

Step 3: Install Essential Extensions

Extensions add new features to your editor. We need two for our course.

  1. Open Cursor.
  2. Click the Extensions icon on the far-left vertical bar (it looks like four squares).
  3. In the search bar that appears, search for and install these two extensions:
    • Python (by Microsoft): This provides core support for the Python language, including better auto-complete and error checking.
    • Quarto (by Quarto): This gives you syntax highlighting and a helpful ‚ÄúRender‚Äù button for our documents.

Step 4: Special Instructions for Windows (WSL) Users

If you are using Windows with WSL (the recommended setup), you must do one more thing to connect Cursor to your Linux environment.

  1. Install the WSL extension: In the Extensions view, search for WSL and install the extension from Microsoft.
  2. Always open projects from the Ubuntu/Linux terminal. This is the crucial part.
The Correct Way to Open Projects on WSL

To make Cursor talk to your Linux files, you must launch it from the Linux (Ubuntu) terminal.

  1. Open your Ubuntu terminal (not PowerShell).
  2. Navigate to your project directory, e.g., cd ~/Documents/your-project-folder.
  3. From inside that folder, type the command cursor . and press Enter.

The . is a shortcut for ‚Äúthis current folder.‚Äù This command opens the Cursor application correctly connected to your Linux environment, allowing it to see your files and use the Linux version of Python you installed.
