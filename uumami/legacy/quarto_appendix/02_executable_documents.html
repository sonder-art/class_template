Chapter 2: Bringing Documents to Life with Code

What is an Executable Document?

This is the feature that makes Quarto magical. An executable document is a file that weaves together three things: 1. Narrative: Your explanatory text, written in Markdown. 2. Code: Chunks of live, runnable code (in our case, Python). 3. Results: The output of that code (like plots, tables, or text), which is automatically generated and inserted into the final document.

This means your document is not just a description of your work; it is your work. It‚Äôs a reproducible record of your analysis that can be easily updated.

Your First Code Chunk

A code chunk is just a special kind of Markdown code block. Instead of three backticks, you use three backticks followed by {python}.

# A regular markdown code block just shows code:
print("This won't be run.")
Show/Hide Code 
# This is a Quarto code chunk.
# The code inside will be executed!
message = "Hello from Python!"
print(message)
Hello from Python!

When you render this document, Quarto will run the Python code in the second block and include its output right below it.

The Most Important Chunk Options

You can control exactly how a code chunk behaves using special comments that start with #|. These are like switches that change how Quarto treats the chunk.

#| echo: false (Hiding the Code)

This is the most common option you‚Äôll use. It tells Quarto to run the code but not to show the code itself in the final document. This is perfect for when you want to show a plot or a table without cluttering the page with the code that generated it.

#| eval: false (Showing Code Without Running It)

This does the opposite. It shows the code block in the document but prevents Quarto from running it. This is useful for writing tutorials (like this one!) where you want to show an example of code without executing it.

Show/Hide Code 
# This code looks runnable, but Quarto will not execute it.
print("You won't see this message in the rendered document.")

#| include: false (Running Code Invisibly)

This option tells Quarto to run the code but to include neither the code nor its output in the final document. This is useful for setup code, like loading libraries or data, that your other code chunks need but that your reader doesn‚Äôt need to see.

#| cache: true (Making Your Document Faster)

As we discussed before, this tells Quarto to save the results of a code chunk after the first time it runs. If the code hasn‚Äôt changed, Quarto will reuse the saved results on future renders instead of re-running the code. This is essential for code that takes a long time to run.
